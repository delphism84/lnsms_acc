# 시리얼 포트 프로토콜 테스트 데이터

이 문서는 Care Receiver와 PC 간 시리얼 포트 통신 프로토콜 테스트를 위한 데이터입니다.
에뮬레이터에서 아래 데이터를 전송하여 각 기능을 테스트할 수 있습니다.

## 프로토콜 개요

### Care Receiver → PC
- **준비완료**: `crcv.ready\r\n`
- **벨 수신코드**: `crcv.bell=0d0af3\r\n` (벨코드 5자리 + 키코드 1자리)
- **장애인 리모컨 수신**: `crcv.assist\r\n`

### PC → Care Receiver
- **통신체크**: `crcv\r\n`
- **응답**: `ok\r\n`

---

## 테스트 케이스 1: 벨 호출 (벨코드: 0d0af, 키코드: 3)

### ASCII 텍스트 형식
```
crcv.bell=0d0af3\r\n
```

### HEX ASCII 형식
```
63 72 63 76 2E 62 65 6C 6C 3D 30 64 30 61 66 33 0D 0A
```

**설명:**
- `63 72 63 76`: "crcv" (ASCII)
- `2E`: "." (ASCII)
- `62 65 6C 6C`: "bell" (ASCII)
- `3D`: "=" (ASCII)
- `30 64 30 61 66`: "0d0af" (벨코드 5자리)
- `33`: "3" (키코드 1자리)
- `0D 0A`: `\r\n` (CRLF)

**예상 동작:** 벨 코드 "0d0af"에 해당하는 문구가 알림창에 표시되어야 합니다.

---

```

```

## 사용 방법

1. 시리얼 포트 에뮬레이터를 실행합니다.
2. 애플리케이션에서 설정한 포트(예: COM1)에 연결합니다.
3. 위의 데이터 중 하나를 선택하여 전송합니다.
   - **텍스트 모드**: ASCII 텍스트 형식을 그대로 복사하여 전송
   - **HEX 모드**: HEX ASCII 형식을 공백 없이 전송 (예: `637263762E62656C6C3D3064306166330D0A`)
4. 애플리케이션의 알림창에서 해당 벨 코드에 매칭된 문구가 표시되는지 확인합니다.

## 참고사항

- 벨 코드는 **5자리**여야 합니다 (예: "0d0af", "12345", "abcde")
- 키 코드는 **1자리**입니다 (예: "1", "2", "3")
- 모든 메시지는 `\r\n` (CRLF, 0x0D 0x0A)로 종료되어야 합니다.
- 벨 코드는 애플리케이션의 "기능 설정 > 문구 관리"에서 등록된 벨 코드와 일치해야 합니다.
- 프로토콜은 대소문자를 구분합니다.

## 추가 테스트 케이스 (선택사항)

### 테스트 케이스 7: 알 수 없는 벨 코드

```
crcv.bell=999991\r\n
```

**설명:** 벨 코드 "99999" (등록되지 않은 코드)
**예상 동작:** 기본 알림 메시지가 표시되거나 무시될 수 있습니다.

### 테스트 케이스 8: 잘못된 형식

```
crcv.bell=123\r\n
```

**설명:** 벨 코드가 5자리 미만
**예상 동작:** 파싱 오류 또는 무시될 수 있습니다.
