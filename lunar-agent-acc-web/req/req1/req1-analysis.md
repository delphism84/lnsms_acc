# req1 요구사항 분석 (최종)

## 개발 구조

- **.NET 백엔드 (CareReceiverAgent.Host)**  
  1. **네이티브 구동**: 시리얼(RS232), 트레이, 알림창(WebView2) 등 Windows 네이티브 기능 담당.  
  2. **웹 서비스**: Kestrel로 웹 서버 구동 → React 빌드 결과물(정적 파일) 서빙 + REST API 제공.

- **데이터 저장소**: **MongoDB** 사용 기준. **JSON 형태, document 형태**로 개발.  
  - 문구(phrases), 시리얼 설정(serial_settings) 등을 문서 단위로 저장.  
  - 연결 정보 없으면 기존 JSON 파일 방식으로 폴백 가능.

---

## 최종 요구사항의 목적

- **호출벨(RF 송신)**과 **리시버(RF 수신 → RS232 → PC)**를 통해 들어온 호출을 **Agent(.NET)**가 수신·관리.
- 수신 데이터(sn/호출코드/암호화ON)를 **고객ID–매장ID 트리**로 구조화하고, **호출벨 1개 ↔ 문구 1개**로 매핑해 알림(문구·색상·이미지) 표시.
- **에이전트설정**을 한 덩어리로 정의. 로그인 시 BE DB로만 동기화(템플릿 **파일** 개념 없음).
- **네트워크등록** 기능은 요구사항에서 제외(스킵·무시).

## 1) 명확한 것

| 구분 | 내용 |
|------|------|
| **호출벨** | RF 송신만. Agent에서 **메이커/모델**을 설정해 구분. (추후 BE에서 업체명/모델명 코드화하여 내려받음) |
| **리시버** | RF 수신만 → RS232 → PC. 포트 이름, 전송속도, 자동인식 설정. |
| **Agent** | .NET 프로그램. 수신 시 **sn / 호출코드 / 암호화ON** 형태로 수신. |
| **조직 구조** | 고객ID – (고객ID 종속) 매장ID 트리. **지금은 로그인 없이 개발.** BE에서 추후 결정. |
| **에이전트설정** | 모든 설정 통칭. **BE에서 DB로만 다룸. 템플릿 파일 개념 없음.** |
| **문구** | 호출벨 1개 : 문구 1개. 문구 **색상·이미지·수정** 가능. |
| **알림** | 여러 번 호출 겹치면 **큐 관리** → 알림창에서 확인. **일괄확인** = 큐 모두 비우고 모두 확인. |
| **수신 모듈** | 수신모듈 시리얼번호·호출벨 설정 필드는 **직접 수정 가능**. |
| **시리얼 자동인식** | **COM 자동 선택.** Baud는 정해짐. COM 선택 후 저장은 테스트 없이 **강제 저장.** 자동검색은 모달 띄우고 취소 가능, 직접 송수신 체크로 자동 검출. |
| **로그인** | 추후 BE 구현. 지금은 무시. (사용자ID+매장ID 키매핑 예정) |
| **공지사항/초기화 ON·OFF** | **지금 무시.** |

## 2) 결정 사항 (질문 답변 반영)

| # | 항목 | 결정 |
|---|------|------|
| 1 | 고객ID–매장ID 트리 | BE에서 결정. **현재는 로그인 없는 걸로 개발.** |
| 2 | 에이전트설정 템플릿 | **BE에서 DB로만 다룸. 템플릿 파일 개념 없음.** |
| 3 | 일괄확인 | **큐 모두 비우고, 모두 확인하는 효과.** (구현 완료) |
| 4 | 호출벨 메이커/모델 | **추후 BE에서 관리.** 메이커/모델 선택 UI 추가구현 |
| 5 | 로그인 | **추후 BE에서 구현. 지금은 무시.** 사용자ID+매장ID로 키매핑 예정. |
| 6 | 수신 모듈 필드 | **직접 수정 가능.** |
| 7 | 시리얼 자동인식 | COM 자동 선택. baud는 정해짐. 저장 시 테스트 없이 강제 저장. 자동검색은 모달에서 취소 가능, 송수신 체크로 자동 검출. |
| 8 | 공지사항/초기화 ON·OFF | **지금 무시.** |

## UI 구성 (반영 완료)

- **설정 화면**: 기능 설정 헤더(뒤로가기, 시리얼 포트 설정, TTS ON/OFF, 닫기) + 문구 관리 카드. 네트워크등록 없음. 카드형·심플 구성.
- **알림창**: 타이틀, 대기 건수 배지, 문구(색상)·이미지, **확인** / **일괄 확인** / 설정. 큐·일괄확인 지원.

---

## 데이터 저장소 (MongoDB 기준)

- **문서 형태**: 기존 JSON 구조와 동일한 document 형태.
  - **phrases** 컬렉션: 한 문서에 문구 목록 전체(PhraseDatabaseDoc).
  - **serial_settings** 컬렉션: 한 문서에 시리얼 설정(SerialSettingsDoc).
- **app.json**에 `mongoConnectionString`이 있으면 MongoDB 사용, 없으면 data/*.json 파일 사용(폴백).
