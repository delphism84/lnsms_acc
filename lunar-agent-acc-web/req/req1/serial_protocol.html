시리얼 프로토콜 · 시리얼번호 · 암호화 (req1-analysis 최종 적용)

리시버(RF 수신) → RS232 → PC Agent. 수신 데이터: sn / 호출코드 / 암호화ON 형태.

1. 최종 프로토콜 (UART v4)

  • 라인 구분: \r (CR). \n은 무시.
  • 형식: <prefix>.<body>. prefix는 8자리 시리얼번호 또는 crcv.
  • 통신 방향: PC → 모듈 (TX), 모듈 → PC (RX).
구분형식설명
통신 체크 (PC→모듈)<sn>\rsn: 8자리 숫자. 모를 때 00000000
통신 체크 응답 (모듈→PC)<sn>.ok\r수신 시 PC가 해당 sn 획득
준비완료 (모듈→PC)<sn>.ready\rCare Receiver 준비완료
도움 요청 (모듈→PC)<sn>.assist\r벨코드 crcv.assist로 처리
벨 코드 (모듈→PC)<sn>.bell=<code>\r호출코드(소문자 정규화)
시드 동기화 (양방향)<sn>.seed=<44자>\r암호화 세션 시드 공유

2. 시리얼번호 (8자리)

Before (설정 전 / 모를 때)

항목값
DeviceSerialNumber (설정)00000000
PC → 모듈 통신 체크00000000\r
모듈 → PC 응답 예12345678.ok\r → PC가 sn 12345678 획득

After (설정 후 / sn 확정)

항목값
DeviceSerialNumber (설정)12345678 (8자리 숫자)
PC → 모듈 통신 체크12345678\r
보안 ON 시 seed 전송12345678.seed=<44자 mark>\r

시리얼번호는 <sn>.ok 수신 시 자동 갱신되며, 로그에 "device serial detected" 기록.

3. 암호화 (선택)

암호화 OFF: 모든 메시지 평문. 암호화 ON: 세션 시드 동기화 후, 페이로드를 16바이트(32-hex)로 인코딩/복호화.

Before (암호화 OFF · 평문)

방향라인 예시
모듈 → PC (도움 요청)12345678.assist\r
모듈 → PC (벨 코드)12345678.bell=qa.1\r
모듈 → PC (통신 체크 응답)12345678.ok\r

After (암호화 ON · 동기화 및 암호문)

단계내용
1) PC → 모듈 시드 전송12345678.seed=<44자 mark>\r (평문). mark 안에 세션 시드(16bit)가 숨겨짐.
2) 모듈 → PC 시드 전송crcv.seed=<44자>\r 또는 <sn>.seed=<44자>\r. PC는 DecodeSeedMarkString으로 시드 복구 후 SecureEnabled=true.
3) 평문 → 암호문 (모듈→PC 예)평문 12345678.bell=qa.1 → body bell=qa.1 (최대 15바이트) + CRC 1바이트 = 16바이트 → 세드 기반 난독화+스왑 → 32-hex. 수신 라인: 12345678.<32hex>\r
4) 암호문 → 평문 (PC 수신 시)12345678.<32hex> 수신 시 TryDecryptLine(sessionSeed) → 12345678.bell=qa.1 로 복원 후 기존 프로토콜처럼 처리.

페이로드: 최대 15바이트 평문 + CRC-8 1바이트 = 16바이트. 인코딩: 세션 시드 기반 XOR/오프셋/스왑 후 32자 hex 문자열.

4. Before / After 요약 예제

상황BeforeAfter
시리얼 번호 00000000 (미확인) 12345678 (<sn>.ok 수신 후 확정)
도움 요청 1건 (평문) 12345678.assist\r 동일 (암호화 OFF 시)
도움 요청 1건 (암호 ON) 평문 12345678.assist RX: 12345678.<32hex>\r → 복호화 → 12345678.assist → assist 처리
벨 코드 호출 (평문) 12345678.bell=qa.1\r 동일 (암호화 OFF 시)
벨 코드 호출 (암호 ON) 평문 body bell=qa.1 RX: 12345678.<32hex>\r → 복호화 → 12345678.bell=qa.1 → bell 코드 처리·알림 큐

5. req1-analysis 반영

  • 수신 형태: sn / 호출코드 / 암호화ON → 위 프로토콜과 시리얼번호·암호화로 수신.
  • 리시버: RF 수신만 → RS232 → PC. 포트·전송속도·자동인식(COM) 설정.
  • 수신 모듈: 시리얼번호·호출벨 설정 필드 직접 수정 가능.

6. 구현 참조 (CareReceiverAgent.Host)

본 문서는 다음 코드 기준으로 정리됨.

  • 프로토콜 파싱: SerialPortBackgroundService.ProcessSerialData (ok, ready, assist, bell=, seed=)
  • 시리얼번호: SerialPortService (DeviceSerialNumber 8자리, <sn>.ok 수신 시 CurrentSerialNumber 갱신)
  • 암호화: SecureSerialCodec (세드 생성/복구, 32-hex 페이로드 암복호, CRC-8)
